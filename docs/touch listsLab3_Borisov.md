# Списки в Markdown

## План разработки

### Как создать репозиторий

#### Шаг 1: Подготовка структуры

1. Создать папку проекта со следующей структурой:
   - Файл `README.md` для описания проекта
   - Папка `src/` для хранения исходного кода
   - Папка `docs/` для документации
   - Файл `.gitignore` для исключения ненужных файлов

#### Шаг 2: Инициализация репозитория

2. Инициализировать Git-репозиторий
   ```bash
   git init
3. Настройка репозитория
   Добавить все файлы в staging area
   ```bash
   git add .
   Проверить статус изменений
   ```bash
   git status
4. Создать первый коммит с описанием
   ```bash
   git commit -m "Initial commit: базовая структура проекта"

#### Шаг 3: Работа с удалённым репозиторием
5. Настройка удаленного репозитория
   ```bash
   git remote add origin <url>
   git push -u origin main
